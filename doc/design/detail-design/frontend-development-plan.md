# GIRA前端开发计划

## 总体概述
- 开发周期：25天
- 技术栈：React 18 + TypeScript + Ant Design
- 开发模式：敏捷开发，每周迭代

## 详细开发计划

### 一、环境搭建与项目初始化 (2天)
1. 环境准备
   - 安装Node.js (v18+)和pnpm
   - 配置开发环境(VSCode + 插件)

2. 项目创建与配置
   - 使用Vite创建React+TS项目
   - 配置项目基础设置(vite.config.ts, tsconfig.json)
   - 安装核心依赖包

### 二、项目基础架构搭建 (3天)
1. 目录结构搭建
   ```
   src/
   ├── assets/       # 静态资源
   ├── components/   # 通用组件
   ├── config/       # 配置文件
   ├── hooks/        # 自定义Hooks
   ├── layouts/      # 布局组件
   ├── pages/        # 页面组件
   ├── services/     # API服务
   ├── store/        # 状态管理
   ├── types/        # 类型定义
   └── utils/        # 工具函数
   ```

2. 核心功能配置
   - 配置Axios请求封装
   - 设置Redux状态管理
   - 实现路由配置
   - 配置权限管理

### 三、认证模块开发 (3天)
1. 登录页面
   - 实现登录表单
   - 集成表单验证
   - 实现Token管理
   - 添加记住密码功能

2. 权限控制
   - 实现路由守卫
   - 配置权限拦截器
   - 处理Token过期

### 四、看板模块开发 (6天)
1. 看板基础功能
   - 看板布局实现
   - 任务卡片组件
   - 看板列组件
   - 看板配置（管理员权限）

2. 拖拽功能
   - 实现任务拖拽
   - 添加WIP限制检查（管理员可配置）
   - 状态更新处理（TODO/IN_PROGRESS/DONE）

3. 筛选与搜索
   - 实现状态筛选
   - 实现经办人筛选（管理员可查看所有，开发者只能查看自己）
   - 实现优先级筛选（LOW/MEDIUM/HIGH）
   - 实现关键字搜索

### 五、Sprint模块开发 (6天)
1. Sprint管理
   - Sprint创建界面（管理员权限）
   - Sprint状态管理（管理员权限）
   - 进度跟踪功能
   - Sprint任务筛选（管理员可查看所有，开发者只能查看自己的任务）

2. 任务池
   - 待办任务列表（管理员可查看所有，开发者只能查看自己的任务）
   - 任务拖拽分配（管理员可分配任务给任何人，开发者只能更新自己的任务）
   - 任务状态和优先级筛选

### 六、组件库开发 (3天)
1. 通用组件
   - 任务卡片组件
   - 状态标签组件
   - 优先级选择器
   - Sprint卡片组件

2. 业务组件
   - 任务详情抽屉
   - Sprint创建表单
   - 看板配置面板

### 七、性能优化与测试 (3天)
1. 性能优化
   - 组件懒加载
   - 状态管理优化
   - 网络请求优化

2. 测试用例
   - 单元测试
   - 集成测试
   - E2E测试

### 八、文档完善与部署 (2天)
1. 文档编写
   - 组件使用文档
   - API接口文档
   - 部署文档

2. 构建与部署
   - 生产环境构建
   - 部署配置
   - 性能监控

## 开发规范

### 1. 代码规范
- 使用TypeScript强类型
- 遵循ESLint规则
- 使用Prettier格式化
- 变量命名采用camelCase
- 组件命名采用PascalCase
- 常量使用大写字母和下划线

### 2. 组件开发规范
- 组件文件结构统一
- Props类型定义完整
- 样式模块化管理
- 组件职责单一
- 复用逻辑提取为Hooks

### 3. 权限管理规范
- 角色分为管理员(ADMIN)和开发者(DEVELOPER)两种
- 权限检查使用usePermission Hook
- 管理员特有权限：
  - 看板配置管理
  - WIP限制设置
  - 查看所有用户任务
  - Sprint创建与管理
- 开发者权限：
  - 查看和更新自己的任务
  - 参与Sprint
  - 更新任务状态

### 4. Git工作流
- 遵循Git Flow流程
- 提交信息规范化
- 代码审查流程
- 分支命名规范
- 版本发布流程

## 技术栈选型
- 核心框架：React 18
- UI组件库：Ant Design 5.x
- 状态管理：Redux Toolkit
- 路由管理：React Router 6
- 类型系统：TypeScript 5.x
- HTTP客户端：Axios
- 样式方案：Less + CSS Modules
- 构建工具：Vite
- 代码规范：ESLint + Prettier
- 测试框架：Jest + React Testing Library
