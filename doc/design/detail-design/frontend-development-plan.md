## 开发原则

1. 只实现文档中包含的功能，不添加任何功能
2. 以页面为单位指定开发顺序
3. 提前完成通用组件的开发
4. 严格按照文档中的前端架构
5. 遵循代码规范和最佳实践
6. 代码复用要求：
   - 以现有代码为基础进行开发
   - 优先复用已有的组件和工具
   - 不随意新增组件，除非确实必要
   - 新增组件前先评估是否可以通过配置或扩展现有组件实现
   - 保持代码结构的一致性和简洁性
